## Overview

The microloader is a minimal payload that runs at boot time, before the Little Kernel (LK) bootloader. Its sole purpose is to load the LK payload from eMMC storage into memory and execute it. It serves as the persistent exploit component that survives reboots.

**Source:** `microloader/main.c:31`

## Memory Layout

### Key Addresses

```c
#define PAYLOAD_DST 0x41000000   // Where LK payload is loaded
#define PAYLOAD_SRC 0x80000       // Primary eMMC offset (512KB)
#define PAYLOAD_SIZE 0x80000      // 512KB
#define BACKUP_SRC 0x200000       // Backup eMMC offset (2MB)

#define BOOT0_PART 1              // eMMC boot partition 1
```

**Implementation:** `lk-payload/common.h:18-25` (shared with LK payload)

<Note>
The microloader is installed in the boot image at offset 0x00-0x400 (1KB). It executes before LK and loads the full LK payload from the eMMC boot partition.
</Note>

## Boot Flow

1. **Device powers on**
2. **BootROM loads boot image** (including microloader)
3. **Microloader executes**
   - Get device structure
   - Read LK payload from eMMC BOOT0 partition
   - If primary payload invalid, try backup location
   - Clean cache
   - Jump to LK payload
4. **LK payload executes**
5. **Normal boot continues**

## Implementation

### Main Function

```c
int main() {
    puts("microloader");
    
    struct device_t *dev = (void*)get_device();
    
    uint32_t *dst = (void*)PAYLOAD_DST;
    size_t ret = dev->read(dev, PAYLOAD_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
    
    // Check if payload is valid
    if (*dst != 0xe3a0d442) {
        puts("Try backup");
        size_t ret = dev->read(dev, BACKUP_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
    }
    
    cache_clean(dst, PAYLOAD_SIZE);
    
    // Jump to the payload
    void (*jump)(void) = (void*)dst;
    puts("Jump");
    jump();
    
    while (1) {}
}
```

**Implementation:** `microloader/main.c:31-58`

## Payload Validation

The microloader validates the payload by checking the first instruction:

```c
if (*dst != 0xe3a0d442) {
    // Invalid - try backup
}
```

**Magic Value:** `0xe3a0d442`

This is the ARM instruction `mov sp, #0x11000000` which should be the first instruction of the LK payload.

**Implementation:** `microloader/main.c:41-44`

<Note>
If the primary payload is corrupted or erased, the microloader automatically falls back to the backup location. This provides redundancy against accidental payload corruption.
</Note>

## Installation

The microloader must be installed in the boot image:

### Boot Image Structure

```
Offset 0x000: Microloader code (max 1KB)
Offset 0x400: Original boot image header
Offset 0x800+: Kernel, ramdisk, etc.
```

The microloader occupies the first 1KB of the boot image, which is normally unused padding before the Android boot image header.

### Installation Process

```python
import struct

def install_microloader(stage2, microloader_bin, lk_payload_bin):
    """
    Install microloader and LK payload to device
    
    Args:
        stage2: Stage2 instance with eMMC access
        microloader_bin: Compiled microloader binary
        lk_payload_bin: Compiled LK payload binary
    """
    # 1. Read original boot partition
    stage2.switch_partition(8)  # USER_PART
    
    # Find boot partition offset from GPT
    boot_start = find_partition_start(stage2, 'boot')
    
    # Read boot image header (skip first 1KB)
    stage2.switch_partition(8)
    boot_blocks = []
    for i in range(256):  # Read 128KB
        boot_blocks.append(stage2.read_block(boot_start + i))
    boot_image = b''.join(boot_blocks)
    
    # 2. Inject microloader
    # Pad microloader to 1KB
    microloader_padded = microloader_bin.ljust(0x400, b'\x00')
    
    # Combine: microloader + original boot image (from offset 0x400)
    modified_boot = microloader_padded + boot_image[0x400:]
    
    # 3. Write modified boot image
    for i in range(len(modified_boot) // 512):
        block_data = modified_boot[i*512:(i+1)*512]
        stage2.write_block(boot_start + i, block_data)
    
    # 4. Write LK payload to BOOT0 partition
    stage2.switch_partition(1)  # BOOT0_PART
    
    # Primary location (0x80000 = block 0x400)
    for i in range(len(lk_payload_bin) // 512):
        block_data = lk_payload_bin[i*512:(i+1)*512].ljust(512, b'\x00')
        stage2.write_block(0x400 + i, block_data)
    
    # Backup location (0x200000 = block 0x1000)
    for i in range(len(lk_payload_bin) // 512):
        block_data = lk_payload_bin[i*512:(i+1)*512].ljust(512, b'\x00')
        stage2.write_block(0x1000 + i, block_data)
    
    print("Microloader and LK payload installed successfully")

# Usage
with open('microloader.bin', 'rb') as f:
    microloader = f.read()

with open('lk-payload.bin', 'rb') as f:
    lk_payload = f.read()

install_microloader(stage2, microloader, lk_payload)

# Reboot device
stage2.reboot()

# Device will now boot with exploit persistence
```

## Error Handling

The microloader has minimal error handling:

- If the primary payload fails validation, it tries the backup
- If both fail, it jumps anyway (likely causing a crash)
- No timeout or recovery mechanism

```c
if (*dst != 0xe3a0d442) {
    puts("Try backup");
    // Try backup, but don't check if it succeeds
    dev->read(dev, BACKUP_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
}

// Jump regardless of validation
jump();
```

**Implementation:** `microloader/main.c:41-51`

<Note>
The microloader is intentionally minimal to fit in 1KB. It relies on the LK payload backup mechanism to maintain redundancy.
</Note>

Typical microloader size is ~250 bytes, leaving ample room for expansion.

## Removal

To remove the exploit:

```python
def remove_microloader(stage2, original_boot_bin):
    """
    Restore original boot image without microloader
    """
    boot_start = find_partition_start(stage2, 'boot')
    
    stage2.switch_partition(8)
    for i in range(len(original_boot_bin) // 512):
        block_data = original_boot_bin[i*512:(i+1)*512]
        stage2.write_block(boot_start + i, block_data)
    
    # Optionally erase LK payload from BOOT0
    stage2.switch_partition(1)
    zero_block = b'\x00' * 512
    for i in range(0x400, 0x400 + 0x100):
        stage2.write_block(i, zero_block)
    
    print("Microloader removed, device will boot normally")
```

## Communication with LK Payload

The microloader can pass flags to the LK payload via memory:

```c
// Before jumping, set flag at known address
uint8_t *flag_addr = (void*)0x41e0062c;
memcpy(flag_addr, "FASTBOOT_PLEASE", 15);

// Then jump
jump();
```

The LK payload checks this address:

```c
uint8_t *tmp = (void*)0x41e0062c;
if (strncmp(tmp, "FASTBOOT_PLEASE", 15) == 0) {
    fastboot = 1;
}
```

**Implementation:** `lk-payload/main.c:109-114`

<Note>
This allows the microloader to influence boot mode without requiring MISC partition access, useful for emergency fastboot access.
</Note>

## Limitations

- **No storage writes**: Cannot update itself or the LK payload
- **No error recovery**: Limited error handling due to size constraints
- **No user interaction**: Cannot present boot menu or accept input
- **Platform-specific**: Hardcoded addresses work only on specific devices

## Advanced Usage

### Conditional Loading

The microloader could be extended to check conditions before loading:

```c
// Check volume buttons
volatile uint32_t *gpio = (volatile uint32_t *)0x10005000;
if (gpio[0x100/4] & 0x02) {  // Vol+ pressed
    // Load alternate payload
    dev->read(dev, ALTERNATE_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
} else {
    // Normal load
    dev->read(dev, PAYLOAD_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
}
```

<Note>
This would enable boot menu functionality directly in the microloader, but increases size and complexity.
</Note>

### Multi-stage Loading

For very large payloads, the microloader could implement two-stage loading:

```c
// Load small stage 2 loader
dev->read(dev, PAYLOAD_SRC, dst, 0x1000, BOOT0_PART);
jump();  // Stage 2 loader loads full payload
```

This allows for payloads larger than the 512KB limit.
