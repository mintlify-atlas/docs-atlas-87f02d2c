## Overview

The payload loader module provides functions to load and execute multi-stage exploit payloads on MediaTek devices. It handles the complete exploit chain from bootrom to stage 2 payload execution.

## Functions

### load_payload(dev)

Loads the complete two-stage bootrom exploit payload.

This function performs the full exploit sequence:
1. Performs bootrom handshake
2. Disables hardware watchdog
3. Exploits bootrom vulnerability via USB control transfer
4. Loads stage 1 payload into bootrom
5. Loads and executes stage 2 payload

<ParamField path="dev" type="Device" required>
  Device instance to load payload on.
</ParamField>

**Raises:** `RuntimeError` if payload fails to load or expected handshake patterns are not received.

**Required files:**
- `../brom-payload/stage1/stage1.bin`
- `../brom-payload/stage2/stage2.bin`

```python
from common import Device
from load_payload import load_payload

dev = Device()
dev.find_device()

if not dev.preloader:
    load_payload(dev)
    # Device now has full payload loaded
    # Can use eMMC operations, memory read/write, etc.
```

### load_pl_payload(dev)

Loads payload when device is in preloader mode.

This simplified loader is used when the device is already in preloader mode instead of bootrom mode. It loads a single-stage payload directly.

<ParamField path="dev" type="Device" required>
  Device instance in preloader mode.
</ParamField>

**Raises:** `RuntimeError` if expected handshake pattern is not received.

**Required files:**
- `../brom-payload/pl/pl.bin`

```python
from common import Device
from load_payload import load_pl_payload

dev = Device()
dev.find_device()

if dev.preloader:
    load_pl_payload(dev)
    # Payload loaded and ready
```

## Helper Functions

### load_payload_file(path)

Loads a binary payload file and pads it to 4-byte alignment.

<ParamField path="path" type="string" required>
  Path to payload binary file.
</ParamField>

<ResponseField name="payload" type="bytes">
  Payload data padded to 4-byte alignment with null bytes.
</ResponseField>

```python
payload = load_payload_file("../brom-payload/stage1/stage1.bin")
log(f"Loaded payload: 0x{len(payload):X} bytes")
```

## Payload Protocol

The payload loader uses a custom protocol after stage 1 is loaded:

### Protocol Format

All commands follow this structure:
```python
magic = 0xf00dd00d  # 4 bytes, big-endian
cmd = 0xXXXX        # 4 bytes, big-endian
# ... command-specific data ...
```

### Command Codes

- `0x1000` - Read eMMC block
- `0x1001` - Write eMMC block
- `0x1002` - Switch eMMC partition
- `0x2000` - Read RPMB
- `0x2001` - Write RPMB
- `0x3000` - Reboot device
- `0x3001` - Kick watchdog
- `0x4000` - Load memory buffer
- `0x4001` - Execute from memory
- `0x5000` - Read memory
- `0x7000` - Read IDME field

## Error Handling

Both payload loading functions validate handshake patterns:

```python
# Stage 1 expects
data = d.read(4)
if data != b"\xA1\xA2\xA3\xA4":
    raise RuntimeError(f"received {data} instead of expected pattern")

# Stage 2 expects
data = d.read(4)
if data != b"\xB1\xB2\xB3\xB4":
    raise RuntimeError(f"received {data} instead of expected pattern")

data = dev.dev.read(4)
```
