## What is Kamakiri?

Kamakiri is a powerful bootrom exploit for MediaTek devices that enables device unlock and custom firmware installation. Originally developed for Amazon Fire tablets (Mantis/Karnak), this exploit leverages vulnerabilities in the MediaTek bootrom to achieve complete control over the device's boot process.

The exploit uses a multi-stage payload system to progressively gain control of the device, ultimately enabling:

- **Bootloader unlock** without manufacturer authorization
- **Custom recovery installation** (TWRP support)
- **RPMB downgrade** for security bypass
- **Fastboot mode access** for advanced operations
- **Direct eMMC read/write** capabilities

## How It Works

Kamakiri exploits vulnerabilities in the MediaTek bootrom to execute unsigned code. The exploit chain consists of:

1. **USB Handshake** - Establish communication with the device in bootrom mode
2. **Stage 1 Payload** - Execute initial payload in bootrom context
3. **Stage 2 Payload** - Load advanced payload with MMC/eMMC drivers
4. **LK Payload** - Hook the Little Kernel bootloader
5. **Microloader** - Persistent payload loader for boot-time modifications

This multi-stage approach allows the exploit to progressively gain more control while working around memory and size constraints at each stage.

## Supported Devices

Kamakiri currently supports Amazon Fire tablets with MediaTek processors, specifically:

- **Amazon Fire HD 8 (7th Gen)** - Mantis (MT8163)
- **Amazon Fire HD 8 (8th Gen)** - Karnak (MT8163)

The exploit targets specific vulnerabilities in the MT8163 bootrom and may be adaptable to other MediaTek devices with similar bootrom implementations.

<Warning>
  **Use at Your Own Risk** - This exploit modifies critical boot components and eMMC partitions. Improper use can result in a bricked device. Always ensure you have proper backups and understand the risks before proceeding.
</Warning>
