## Payload Architecture

Kamakiri uses a multi-stage payload architecture where each stage enables capabilities for the next. This design is necessary due to size constraints and progressive access to hardware peripherals.

## Stage 1: BROM Payload

### Overview

**Source:** `brom-payload/stage1.c`

**Binary:** `stage1.bin`

**Load Address:** Injected into BROM SRAM via vulnerability

**Size Limit:** 0xA00 (2560) bytes maximum

**Execution Context:** BootROM address space with SRAM access

```c
    char idme_buf[0x400] = { 0 };
    char field_name[16] = { 0 };
    recv_data(field_name, 16, 0);
    printf("Read %s from IDME\n", field_name);

    uint32_t block = 0;

    while (!(type_offset = memmem(idme_buf, 0x400, &field_name, 16)) 
           && block < 0x2000) {
        if (mmc_read(host, block++, buf) != 0) {
            send_dword(4);
            send_dword(0xffffffff);  // Read error
            break;
        }

            send_dword(4);
            send_dword(0xbeefdeed);  // Invalid IDME
            break;

        // Sliding window search
        memcpy(idme_buf, idme_buf + 0x200, 0x200);
        memcpy(idme_buf + 0x200, buf, 0x200);

    if (type_offset) {
        uint32_t len = *(uint32_t*)(type_offset + 16);
        type_offset += 16 + 12;  // Skip to data
        send_dword(len);
        send_data(type_offset, len);

        send_dword(0xdeadbeef);  // Field not found
    }
```

## Preloader Payload (PL)

### Overview

**Source:** `brom-payload/pl.c`

**Binary:** `pl.bin`

**Load Address:** `0x40001000`

**Purpose:** Alternative entry point when device is in preloader mode instead of BROM mode

### Function Discovery via Pattern Matching

Since preloader location may vary, the payload searches for functions by pattern:

```c
#define PRELOADER_BASE 0x201000

uint16_t send_dword_pattern[]  = {0xB507, 0x0E03};
uint16_t recv_dword_pattern[]  = {0x4B0E, 0x2200};
uint16_t msdc_init_pattern[]  = {0xB538, 0x4604, 0x4D0F};

uint32_t searchfunc(uint32_t startoffset, uint32_t endoffset, 
                    uint16_t *pattern, uint32_t patternsize) {
    uint32_t matched = 0;
    for (uint32_t offset = startoffset; offset < endoffset; offset += 2) {
        for (uint32_t i = 0; i < patternsize; i++) {
            if (((uint16_t *)offset)[i] != pattern[i]) {
                matched = 0;
                break;
            }
            if (++matched == patternsize) return offset;
        }
    }
    return 0;
}
```

## LK Payload

### Overview

**Source:** `lk-payload/main.c`

**Binary:** `payload.bin`

**Install Location:** Boot0 partition at offset `0x80000` (512KB)

**Load Address:** `0x41000000` (loaded by microloader)

**Execution Context:** Called before LK boots, has full LK context access

### LK Payload Purpose

The LK payload hooks the bootloader to:
1. Parse GPT for partition locations
2. Manipulate boot mode flags
3. Hook the read() function to hide microloader
4. Patch security checks
5. Enable fastboot/recovery access

```c
static void parse_gpt() {
    uint8_t raw[0x800] = { 0 };
    struct device_t *dev = get_device();
    dev->read(dev, 0x400, raw, sizeof(raw), USER_PART);
    
    for (size_t i = 0; i < sizeof(raw) / 0x80; ++i) {
        uint8_t *ptr = &raw[i * 0x80];
        uint8_t *name = ptr + 0x38;
        uint32_t start;
        memcpy(&start, ptr + 0x20, 4);
        
        if (memcmp(name, "b\x00o\x00o\x00t\x00\x00\x00", 10) == 0) {
            printf("found boot at 0x%08X\n", start);
            g_boot = start;
        } else if (memcmp(name, "r\x00\x65\x00\x63\x00o\x00v\x00\x65\x00r\x00y\x00\x00\x00", 18) == 0) {
            printf("found recovery at 0x%08X\n", start);
            g_recovery = start;
        } else if (memcmp(name, "l\x00k\x00\x00\x00", 6) == 0) {
            printf("found lk at 0x%08X\n", start);
            g_lk = start;
        } else if (memcmp(name, "M\x00I\x00S\x00\x43\x00\x00\x00", 10) == 0) {
            printf("found misc at 0x%08X\n", start);
            g_misc = start;
        }
    }
}
```

```c
size_t read_func(struct device_t *dev, uint64_t block_off, 
                void *dst, uint32_t sz, uint32_t part) {
    printf("read_func hook\n");
    int ret = 0;
    
    // Hook reads to boot or recovery partition
    if (block_off == g_boot * 0x200 || block_off == g_recovery * 0x200) {
        printf("demangle boot image - from 0x%08X\n", 
               __builtin_return_address(0));

        // Redirect to recovery if y-cable connected
        if (block_off == g_boot * 0x200 && *y_cable == 1 
            && !boot_system && *boot_reason == 0) {
            printf("Boot into recovery to display bootmenu");
            block_off = g_recovery * 0x200;
        }
        
        // Skip microloader (first 0x400 bytes)
        if (sz < 0x400) {
            ret = original_read(dev, block_off + 0x400, dst, sz, part);
        } else {
            void *second_copy = (char*)dst + 0x400;
            ret = original_read(dev, block_off, dst, sz, part);
            memcpy(dst, second_copy, 0x400);  // Copy real header
            memset(second_copy, 0, 0x400);     // Clear microloader
        }
    } else {
        ret = original_read(dev, block_off, dst, sz, part);
    }
    return ret;
}
```

## Microloader

### Overview

**Source:** `microloader/main.c`

**Binary:** `microloader.bin`

**Install Location:** Boot partition, first 0x400 bytes (1024 bytes)

**Purpose:** Minimal stage that loads LK payload before LK boots

### Implementation

```c
int main() {
    puts("microloader");

    struct device_t *dev = (void*)get_device();

    uint32_t *dst = (void*)PAYLOAD_DST;  // 0x41000000
    
    // Try to read payload from primary location
    size_t ret = dev->read(dev, PAYLOAD_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
    // PAYLOAD_SRC = 0x80000, PAYLOAD_SIZE = 0x80000 (512KB)

    // Check if payload is valid (magic: 0xe3a0d442)
    if(*dst != 0xe3a0d442) {
        puts("Try backup");
        // Try backup location
        size_t ret = dev->read(dev, BACKUP_SRC, dst, PAYLOAD_SIZE, BOOT0_PART);
        // BACKUP_SRC = 0x200000
    }

    cache_clean(dst, PAYLOAD_SIZE);

    // Jump to the payload
    void (*jump)(void) = (void*)dst;
    puts("Jump");
    jump();  // Transfer control to LK payload at 0x41000000

    while (1) { }  // Should never reach here
}
```

### Microloader Magic

The microloader checks for ARM instruction `0xe3a0d442`:

```assembly
mov sp, #0x42000000  ; Set stack pointer
```

This is the first instruction of the LK payload, serving as a validity check.

### Installation Process

The microloader replaces the Android boot image header:

```python
# Read original boot header (0x400 bytes across 2 blocks)
boot_hdr1 = dev.emmc_read(gpt["boot"][0]) + dev.emmc_read(gpt["boot"][0] + 1)
boot_hdr2 = dev.emmc_read(gpt["boot"][0] + 2) + dev.emmc_read(gpt["boot"][0] + 3)

# Write microloader to first 0x400 bytes
flash_binary(dev, "../bin/microloader.bin", gpt["boot"][0], 2 * 0x200)

# If second header wasn't the kernel (ANDROID! magic), restore original header
if boot_hdr2[0:8] != b"ANDROID!":
    flash_data(dev, boot_hdr1, gpt["boot"][0] + 2, 2 * 0x200)
```

## Payload Size Constraints

| Payload | Max Size | Reason |
|---------|----------|--------|
| Stage1 | 0xA00 (2560 bytes) | BROM SRAM limit |
| Stage2 | ~32KB | SRAM available at 0x201000 |
| PL | ~32KB | Same as Stage2 |
| LK Payload | 512KB | Boot0 allocation |
| Microloader | 1024 bytes | Must fit in boot header |

## Payload Communication Flow

```
Python → [USB Serial] → Stage1 → [Memory Write] → Stage2
                          ↓                          ↓
                     [Confirmation]            [eMMC Access]
                      0xA1A2A3A4                0xB1B2B3B4
                                                     ↓
                                              [Install Payloads]
                                                     ↓
                                              [Reboot Device]
                                                     ↓
                        Microloader → LK Payload → Hooked LK → Kernel
```

## Debugging Output

All payloads output debug information via UART:

```c
void low_uart_put(int ch) {
    volatile uint32_t *uart_reg0 = (volatile uint32_t*)0x11003014;
    volatile uint32_t *uart_reg1 = (volatile uint32_t*)0x11003000;

    while ( !((*uart_reg0) & 0x20) )  // Wait for TX ready
    {}

    *uart_reg1 = ch;  // Send character
}
```

**UART Configuration:**
- Base Address: `0x11003000`
- Baud Rate: 115200 (typically)
- Line Status Register: `+0x14` (bit 5 = TX empty)
- TX Holding Register: `+0x00`

Connect a USB-to-serial adapter to view payload execution in real-time.
